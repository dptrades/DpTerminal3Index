Daily Automated Workflow

Every weekday at 10:00 AM ET, the system executes this sequence automatically via Vercel cron. Zero human intervention required.

10:00:00 AM — Market Check
System wakes up, checks if market is open
Fetch VIX level from Polygon.io
Fetch SPY price and compute 200-day EMA
Decision: If VIX > 30 AND SPY < 200 EMA → CASH MODE (skip all entries, only process exits)
10:00:05 AM — Exit Processing
Check all open positions for exits
For each position, fetch current price from Polygon.io:
• Price ≤ stop-loss → SELL (stop hit)
• Price ≥ entry + 1x ATR (scale level 0) → SELL 30% (scale 1), move stop to breakeven
• Price ≥ entry + 2x ATR (scale level 1) → SELL 30% (scale 2), move stop up
• Scale level 2+ → Update trailing stop (1.5x ATR below high water mark)
• Days held ≥ 30 → SELL ALL (time exit)

Each sell → Robinhood MCP places market sell order on agentic account
10:00:30 AM — Discovery Scan
Scan 300+ stocks from Top Picks + Alpha Hunter
Run scanConviction() → 200+ mega-cap stocks scored on 4 factors
Run scanAlphaHunter() → 300+ growth/momentum stocks scored
Merge and deduplicate → ~400 unique candidates
Data sources: Polygon.io (prices, fundamentals), Yahoo Finance (analyst ratings)
10:01:00 AM — Technical Scoring
Score each candidate using confluence scorer
For each of ~400 stocks, compute:
• EMA stack (9/21/50/200) alignment
• RSI momentum + divergence
• MACD crossover
• Bollinger Band position
• Candlestick patterns
• 52-week high proximity
• Relative strength vs SPY
Result: Technical score 0-100 for each stock
10:02:00 AM — Smart Money Check (NEW)
Query Unusual Whales for institutional confirmation
For top-scored candidates (score ≥ 60, BULLISH trend):
• UW MCP: Check unusual options flow → large call buying = conviction boost
• UW MCP: Check dark pool activity → block accumulation = conviction boost
• UW MCP: Check congressional trades → political insider buying = flag
Adjust conviction score: +5 to +15 for strong smart money confirmation
10:02:30 AM — Risk Filtering
Apply all risk gates before entry
Filter candidates through:
• Score ≥ 60 (conviction threshold)
• Trend = BULLISH only
• Not in symbol blacklist (<25% WR over 20 trades → 30-day suspension)
• Positions < 6 max
• Sector count < 2 per sector
• Not in earnings red zone (0-7 days before earnings)
• Sufficient cash available
Result: Top 1-6 qualifying candidates ranked by score
10:03:00 AM — Order Execution
Place buy orders on Robinhood via MCP
For each qualifying candidate:
1. Calculate position size: 1.5% of equity ÷ (entry - stop) = shares
2. Cap at available cash
3. Robinhood MCP → place market buy order
4. Record entry price, stop-loss, scale levels, ATR
5. Log signal with full audit trail

Simultaneously: Forward test records same decisions with simulated fills (for comparison)
10:03:30 AM — Save & Sync
Persist state, push to GitHub
Save state to Vercel Blob (positions, trades, signals, equity curve)
Commit data files to GitHub for version tracking
Send notification summary (optional email via Resend)
10:04:00 AM — Done
System goes idle until next trading day
Total execution time: ~3-4 minutes
Next run: Tomorrow 10:00 AM ET (weekdays only)
All dashboards updated and viewable at any time
